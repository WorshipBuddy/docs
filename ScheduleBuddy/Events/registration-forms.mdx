# Event Registration Forms

Registration forms allow you to collect information from event attendees. You can create fully customized forms with unlimited questions of various types.

## Accessing Registration Form Builder

To build or edit a registration form:

1. Open your event (click on it from the Services page)
2. Click the **"Event Registration"** tab
3. The form builder will appear with:
   - Settings section
   - Question types library
   - Form preview
   - Your existing questions

## Registration Settings

Before building questions, configure your registration settings:

### Basic Settings

- **Ask about team availability** - Include questions about which teams attendees can serve on
- **Allow multiple registrations** - Let people register multiple times (for registering others)
- **Allow editing responses** - Let registrants edit their responses after submitting
- **Allow anonymous submissions** - Allow registrations without requiring name/email

### Registration Dates

Control when registration is open:

- **Registration Open** (optional) - Registration form won't be accessible before this date/time
- **Registration Soft Close** (optional) - A date shown on the form (informational only)
- **Registration Hard Close** (optional) - Registration form closes after this date/time

<Info>
**Tip**: Set an open date to prevent early registrations. Set a hard close date to automatically stop accepting registrations when you're full or past the deadline.
</Info>

### Additional Services

Enable optional services that will appear as separate tabs:

- **Housing Assistance** - Enable housing management for the event
- **Transportation Assistance** - Enable transportation management
- **Meals Assistance** - Enable meal planning and attendance tracking

<Info>
**Note**: When you enable these services, they'll appear as separate management tabs in the event dashboard, and template questions will be available for each service.
</Info>

## Question Types

You can add various types of questions to your form:

### Text Input
- **Text** - Short text answer
- **Long Text** - Multi-line text answer
- **Email** - Email address input
- **Phone** - Phone number input
- **Address** - Full address input

### Numbers and Dates
- **Number** - Numeric input
- **Date** - Date picker
- **Time** - Time picker
- **Date & Time** - Combined date and time

### Selection
- **Dropdown** - Single selection from a list
- **Multiple Choice** - Single selection from radio buttons
- **Checkboxes** - Multiple selections

### Rating and Scale
- **Rating** - Star rating (typically 1-5)
- **Scale** - Numeric scale with optional labels

### Yes/No
- **Yes/No** - Simple yes or no question

## Adding Questions

### Adding a Question

1. In the form builder, find the question type you want to add
2. Click the question type button (e.g., "Text", "Dropdown", etc.)
3. The question will appear in your form builder
4. Configure the question:
   - **Question Text** - What you're asking
   - **Required** - Whether the question must be answered
   - **Placeholder** (optional) - Example text in the input field
   - **Help Text** (optional) - Additional instructions shown below the question

### Configuring Question Options

For questions that need options (Dropdown, Multiple Choice, Checkboxes):

1. Click **"Add Option"** or edit existing options
2. Enter each option value
3. Options appear as choices in the form

### Question Settings

Each question has additional settings accessible by clicking **"Settings"**:

- **Required** - Make the question mandatory
- **Placeholder** - Example text shown in empty fields
- **Help Text** - Additional instructions
- **Conditional Logic** - Show this question only if another question meets certain criteria

### Conditional Logic

Show questions based on answers to other questions:

1. Click **"Settings"** on a question
2. Enable **"Conditional Logic"**
3. Configure:
   - **Show if** - Which question to check
   - **Condition** - How to check it (equals, contains, greater than, etc.)
   - **Value** - What value to check for

<Info>
**Tip**: Use conditional logic to show relevant questions. For example, show "Dietary restrictions" only if someone answers "Yes" to "Do you have dietary restrictions?"
</Info>

## Reordering Questions

Questions can be reordered:

1. Click and drag the drag handle (⋮⋮) on the left side of a question
2. Drop it in the desired position
3. Questions are saved in the order you arrange them

## Previewing Your Form

To see how your form will look to registrants:

1. Click **"Show Preview"** button
2. A preview panel appears showing the form as registrants will see it
3. Click **"Hide Preview"** to return to the builder

<Info>
**Tip**: Use preview to test your form before publishing. Make sure all questions make sense and the flow is logical.
</Info>

## Saving Your Form

1. Build your questions and configure settings
2. Click **"Save Registration Form"** button
3. Once saved, you can share the registration link

<Warning>
**Warning**: Your form must be saved before you can share the registration link or accept registrations.
</Warning>

## Sharing Registration Links

After saving your form:

1. Click **"Share Registration Link"** button
2. Copy the registration link
3. Share it via email, social media, website, etc.

<Info>
**Note**: The registration link is only available after you've saved your registration form at least once. The link works as long as the event is published and registration dates allow it.
</Info>

## Template Questions

When you enable Housing, Transportation, or Meals assistance, template questions are available:

### Housing Template Questions
- Pre-configured questions for housing needs
- Can add to your form or customize
- Includes questions about accommodation preferences

### Transportation Template Questions
- Questions for transportation needs
- Includes flight information, pickup needs, etc.
- Can customize or add custom transportation questions

### Meals Template Questions
- Questions about dietary needs
- Meal preferences and restrictions
- Can customize or add custom meal questions

<Info>
**Tip**: Template questions provide a good starting point. You can use them as-is, modify them, or create your own custom questions for these services.
</Info>

## Best Practices

1. **Start simple** - Begin with basic questions, add more as needed
2. **Test the form** - Use preview to check how questions look and flow
3. **Make required fields clear** - Only require information you truly need
4. **Use conditional logic** - Reduce form length by showing questions only when relevant
5. **Set registration dates** - Control when people can register
6. **Save frequently** - Save your form as you build it to avoid losing work

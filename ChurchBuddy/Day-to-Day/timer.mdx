# Service Timer

The Timer feature lets you create countdown timers for services with a rundown of timed blocks. Share the display URL with your production team or display it on screens.

## Overview

The Timer provides:

- **Rundown Management** - Create a list of timed blocks for your service
- **Countdown Display** - Full-screen countdown view for display on screens
- **Color Indicators** - Visual warnings as time runs low
- **Shareable Display** - Separate display URL for stage monitors or production

## Creating a Timer

1. Go to **`/timer/`** directly (there is no Timer link in org navigation)
2. Name your timer (e.g., "Sunday Service", "Youth Night")
3. Add blocks to your rundown
4. Share the display link with your team

You can also click **Launch Timer** on a service detail page to open `/timer/` with that service's schedule blocks pre-loaded.

## Building a Rundown

A rundown is a list of blocks that make up your service. Each block has:

- **Name** - What this block is (e.g., "Worship", "Announcements", "Message")
- **Duration** - How long the block should last (minutes only)
- **Notes** - Optional notes for the production team

### Adding Blocks

1. Click **+ Add Block**
2. Enter the block name
3. Set the duration in minutes
4. Add any notes (optional)
5. Repeat for all blocks

### Reordering Blocks

Blocks cannot be reordered by drag-and-drop in the current UI — add blocks in the order you need.

### Editing Blocks

**Double-click** a block to edit its name, duration, or notes. Single click selects it.

## Running the Timer

### Starting the Timer

1. Click the first block to select it
2. Click **Start** or press the play button
3. The timer counts down from the block duration

### Timer Controls

- **Start/Pause** - Start or pause the current countdown
- **Reset** - Stops the timer and clears block progress/selection (confirmation dialog); select a block and **Start** again to run
- **Next** - Move to the next block
- **Prev** - Go back to the previous block

### Color Indicators

The timer display changes color based on remaining time:

| Color | Meaning |
|-------|---------|
| **Green** | On time — plenty of time remaining |
| **Yellow** | Warning — at/below yellow threshold (default 5 min remaining) |
| **Red** | Critical — at/below red threshold (default 2 min remaining) |
| **Flashing red** | Overtime — block duration has expired |

## Display View

The display view shows a clean, full-screen countdown for stage monitors or displays.

### Getting the Display URL

1. Click **Open Display** to open the display view in a new tab, or **Share** to copy the display link from the Share Timer modal
2. Copy the display URL if needed
3. Open on another device or browser window

### Display Features

- Full-screen countdown
- Current block name
- Next block preview
- Color-coded time indicators

Advance blocks from the **controller** page using **Next** — the display view is read-only (no block controls on the display URL).

<Info>
**Tip**: Open the display view on a separate monitor or iPad for your stage display, while controlling the timer from your main screen.
</Info>

## Timer Settings

### Warning Times

Configure when the timer changes to yellow (warning) and red (critical) — thresholds are in **minutes**:

- **Yellow Warning** — Minutes remaining to show yellow (default: **5** minutes)
- **Red Critical** — Minutes remaining to show red (default: **2** minutes)

Advance blocks manually with **Next** / **Prev** — there is no auto-advance setting.

## Use Cases

### Service Planning

1. Create your service rundown in advance
2. Set realistic durations for each block
3. Share with worship leaders and speakers
4. Use during service to stay on schedule

### Production Team

1. Display the timer on stage monitors
2. Production team controls from the booth
3. Speakers and worship leaders see remaining time
4. Keep services running on schedule

### Rehearsals

1. Run through your service with timed blocks
2. Identify blocks that need more or less time
3. Adjust durations based on actual run times

## Best Practices

1. **Add buffer time** - Include transition blocks between main items
2. **Test your display** - Check the display view before service
3. **Communicate** - Make sure all participants know the timer is running
4. **Be flexible** - The timer is a guide, not a strict rule
5. **Session-based timers** — Timer sessions expire after ~24 hours; use **Launch Timer** from the service again or keep controller/display links for the current session

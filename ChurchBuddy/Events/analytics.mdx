# Event Analytics

The Analytics module on the Event Dashboard provides registration trends, response breakdowns, and summary statistics to help you track event progress.

<Info>
**Platform**: Web only.
</Info>

## Accessing Analytics

1. Open the **Event Dashboard** for your event
2. Open the **Analytics** module

## What's Available

The Analytics module typically shows:

- **Registration totals** - Total registrants over time
- **Registration pace** - How quickly registrations are coming in
- **Response breakdowns** - Summary data from registration form questions
- **Trend charts** - Visual registration trends approaching the event date

<Info>
Available charts and metrics depend on your event's registration form configuration and how many responses have been collected.
</Info>

## Using Analytics

### Before the Event

- Monitor registration pace — are you on track?
- Identify popular sessions or options from form responses
- Decide when to close registration or send reminders

### During Setup

- Verify registration form is collecting expected data
- Check that logistics questions (housing, transportation, meals) are being answered

## Related Features

- **Registration responses** — Detailed per-registrant data in [Registration Responses](/ChurchBuddy/Events/registration-responses)
- **CSV export** — Full data export from the Registration module
- **Overview** — Quick stats and setup progress on the Event Dashboard

## Best Practices

1. **Check analytics weekly** as the event approaches
2. **Compare to past events** if you run similar events regularly
3. **Use data for logistics planning** — housing and transportation needs scale with registration numbers
4. **Export for deeper analysis** — Use CSV export for custom spreadsheet analysis

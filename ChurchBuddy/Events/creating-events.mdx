# Creating Events

Events are powerful tools for organizing special gatherings that need more than just scheduling. Whether it's a multi-day conference, a weekend retreat, a workshop, or any special gathering, events provide registration forms, check-in systems, and logistics management all in one place.

<Info>
**Platform**: Event creation and full admin are on the **web app**. The mobile app can view events and open **Operations** for users with event dashboard permissions (Owner, Org Admin, or granular services/registration/logistics access), but creating events requires the web.
</Info>

## Creating an Event

Events now have their own top-level **Events** page in the organization navigation.

1. Click **Events** in the top navigation
2. Click **+ Create Event**
3. Fill in the event information:
   - **Event Name** (required)
   - **Start Date** (required)
   - **End Date** (required)
   - **Location** (optional)
4. Click **Create Event**

You'll be taken straight to the **Event Dashboard** for the new event. New events are created as **Published** status with **Private** visibility — you can change either in the event's **Settings**.

<Info>
Event creation is intentionally lightweight — there are no event "templates" to pick. Create the event first, then turn on only the pieces you need (registration questions, services, and logistics modules) from the dashboard.
</Info>

You can also create an event from **Calendar → Add Event** (Owners and Organization Admins); it collects the same name, dates, and location.

### Enabling logistics modules

Logistics modules (Transportation, Housing, Meals, SignUps) are off by default. Turn them on from the event's **Overview**:

1. Open the event's **Event Dashboard**
2. On **Overview**, find the **Add a module** card
3. Click **Transportation**, **Housing**, **Meals**, or **SignUps**

Changes save automatically and the module appears in the dashboard's module rail. See [Event Dashboard](/ChurchBuddy/Events/event-dashboard).

## Editing Event Details (Owner / Org Admin only)

1. Click **Events** and open your event
2. Open **Settings** in the dashboard's module rail
3. Modify event name, dates, location, status, or visibility
4. Save your changes

### Event Status

- **Draft** - Still being set up; not open for public registration
- **Published** — Event is live (default on create). Public registration requires a saved registration form; open/close dates are optional (with neither set, registration stays open while published).
- **Active** - Event is currently happening
- **Completed** - Event has finished
- **Archived** - Historical reference

### Event Visibility

- **Private** - Only visible to organization members
- **Public** - Registration link can be shared externally

<Info>
New events default to **Published** status and **Private** visibility. Change to Draft if you want to finish setup before opening registration.
</Info>

## Deleting an Event (Owner / Org Admin only)

1. Open the **Event Dashboard**
2. Open **Settings** in the module rail
3. Click **Delete Event**
4. Confirm the deletion

<Warning>
Deleting an event cannot be undone. This removes the event, all registrations, event services, and all logistics data.
</Warning>

## Best Practices

1. **Plan ahead** - Create events early for registration and logistics setup time
2. **Turn on only what you need** - Enable logistics modules from **Add a module** as the event takes shape
3. **Set dates carefully** - Verify start and end dates
4. **Review default status** - Events are Published by default; switch to Draft if setup isn't ready
5. **Use public visibility** - For events open to people outside your organization

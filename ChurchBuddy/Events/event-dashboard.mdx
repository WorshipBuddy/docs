# Event Dashboard

The Event Dashboard is the central hub for managing an event. It opens at `/event-dashboard/?org={org_id}&event={event_id}` when you open an event from the **Events** page.

## Accessing the Event Dashboard

1. Click **Events** in the top navigation
2. Click your event's card
3. The Event Dashboard opens with its module rail

## The Module Rail

The dashboard is organized as a **module rail** — a list of workspaces you switch between. Modules appear based on what's enabled for the event and your permissions.

| Module | Purpose |
|--------|---------|
| **Overview** | Event summary, stat cards, quick actions, and the **Add a module** card |
| **Registration** | Form builder, registrant list, CSV export |
| **Schedule** | Event services, team scheduling, public schedule sharing |
| **Transportation** | Vehicles, drivers, routes, assignments, live flight tracking, live vehicle map, QR check-in |
| **Housing** | Locations, host assignments, Smart Auto-Assign, bulk assign, host dashboards |
| **Meals** | Meal schedules, menus, dietary/allergy tracking, headcounts, live service, crowd meter |
| **SignUps** | Volunteer slot and item/meal sign-up sheets with templates, sharing, and a check-in kiosk |
| **Check-in** | Check-in tools, logs, and scanner links |
| **Analytics** | Registration trends and event statistics |
| **Share** | The **Share Center** — one hub of copyable public links: registration link, event/flight dashboards, crowd meter, housing registration form, and the registration/transportation scanners |
| **Settings** | Event configuration (**Owner / Org Admin only**) |

**Core** modules (Overview, Registration, Schedule, Analytics, Share, Settings) are always present. The **logistics** modules (Transportation, Housing, Meals, SignUps) only appear once enabled. **Check-in** is available without enabling.

<Info>
Modules are filtered by granular permissions. Enabling a logistics module is **Owner / Org Admin only**; **Settings** is only shown to Owners and Org Admins.
</Info>

## Enabling Logistics Modules

Turn logistics modules on from the **Overview**:

1. Open the **Add a module** card on Overview
2. Click **Transportation**, **Housing**, **Meals**, or **SignUps**

Changes save automatically and the module appears in the rail. To turn a module off again, use the same card.

## Overview

The Overview shows:

### Event Header
- **Event Name**, **Dates**, **Location**, and a **Status Badge** (Draft, Published, Active, Completed, or Archived)

### Quick Stats

Stat cards show counts such as:

- **Registrations** and **Total Registrants**
- **Services**
- **Need Transportation**, **Need Housing**, **Need Meals** (from registration answers)

Registration trends and charts live on the **Analytics** module.

### Add a module

The **Add a module** card enables or disables the logistics modules for this event (Owner / Org Admin only).

## Registration

Two areas within Registration:

### Form Builder
- Build and edit the registration form
- Configure registration settings (dates, multiple registrations, editing)
- Add logistics template questions (available once the matching logistics module is enabled)
- Share the registration link via **Share Link**

### Registrations
- View all registration responses
- Export to CSV
- Toggle visible columns with **Columns**

## Schedule

- View all services in the event
- **Schedule Teams** on each service card — opens bulk scheduling filtered to this event's services (**Owner / Org Admin only**)
- Add, edit, or remove event services (**Owner / Org Admin only**)
- Share service schedules publicly

<Info>
Event services don't appear in **Calendar → Services**. They are managed only within this event.
</Info>

## Analytics

View registration trends, response breakdowns, and event statistics — useful for tracking registration pace as the event approaches.

## Settings

Open **Settings** in the rail to modify event details, registration settings, status, and visibility, or to delete the event. Owners and Organization Admins only.

## Event Status Workflow

1. **Draft** - Event is still being set up
2. **Published** — Event is live (default on create). Public registration requires a saved registration form; open/close dates are optional (with neither set, registration stays open while published).
3. **Active** - Event is currently happening
4. **Completed** - Event has finished
5. **Archived** - Event is archived for historical reference

## Mobile App

Users with **event dashboard permissions** (Owner, Org Admin, or granular services/registration/logistics access — not analytics-only or signups-only) can open **Operations** for an event.

## Best Practices

1. **Use the module rail** - Switch between workspaces as you manage the event
2. **Enable only what you need** - Add logistics modules from **Add a module**
3. **Check regularly** - Review registrations and logistics as the event approaches
4. **Update status** - Keep event status current (draft → published → archived)
5. **Export data** - Regularly export registration data for backup

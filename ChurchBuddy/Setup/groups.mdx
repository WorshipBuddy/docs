# Groups

Groups allow you to organize people in your organization for targeted communication, management, and coordination. You can create manual groups by selecting specific people and teams, or automatic groups that dynamically include people based on rules.

## Overview

Groups in ChurchBuddy help you:

- **Organize Members** - Group people by role, team, age, or other criteria
- **Push notifications** - Send mobile push to group members
- **Automatic Management** - Use rules to automatically include/exclude people
- **Flexible Organization** - Combine manual selection with automatic rules

## Group Types

ChurchBuddy supports two types of groups:

### Manual Groups

Manual groups are created by explicitly selecting:
- **Individual Users** - Select specific people by email
- **Teams** - Include all members of selected teams

Manual groups require you to manually add or remove members when your organization changes.

### Automatic Groups

Automatic groups use rules to dynamically determine membership. People are automatically included or excluded based on criteria you define. Rules are evaluated each time the group is accessed, so membership updates automatically as people's information changes.

<Info>
**Tip**: Automatic groups are great for categories that change frequently, like "All Youth" (age-based) or "All Worship Team Members" (position-based). Manual groups work well for fixed committees or special teams.
</Info>

## Creating a Group

To create a new group:

1. Click **People** in the top navigation
2. Click the **Groups** sub-tab
3. Click **Create Group**
4. Enter a **Group Name** (required)
5. Optionally add a **Description**
6. Choose **Group Type**:
   - **Manual Group** - Select people and teams manually
   - **Automatic Group** - Use rules to determine membership
7. Configure the group based on type (see sections below)
8. Click **Create Group** (web) or **Create** (mobile)

<Info>
Groups is not in the top navigation bar. Access it via **People → Groups**. On mobile: **More → Groups** (Owner/Admin only).
</Info>

## Manual Groups

### Adding Members to Manual Groups

For manual groups, you can add members in two ways:

#### Individual Users

1. In the **"Add Users"** section, check the boxes next to people you want to include
2. All checked users will be added to the group

#### Teams

1. In the **"Add Teams"** section, check the boxes next to teams you want to include
2. All members of selected teams will be included in the group
3. If someone is on multiple selected teams, they'll only appear once in the group

<Info>
**Note**: When you add a team to a manual group, all current team members are included. Team-linked membership **updates automatically** when team permissions change — only individually selected users in **User Emails** are static until you edit the group.
</Info>

### Editing Manual Groups

To add or remove members from a manual group:

1. Open the group
2. Click **"Edit"**
3. Check or uncheck users/teams as needed
4. Click **Update Group** (web) or **Save** (mobile)

## Automatic Groups

Automatic groups use rules to determine membership. All rules are combined with **AND logic**, meaning a person must match **ALL rules** to be included in the group.

### Creating Rules

When creating an automatic group, you'll add one or more rules. Each rule has three parts:

1. **Field** - What to check (age, gender, position, team, etc.)
2. **Operator** - How to compare (equals, greater than, has, etc.)
3. **Value** - What to compare against

### Available Fields

You can create rules based on:

- **Age** - Person's age
- **Gender** - Male or Female
- **Marital Status** - Single, Married, Divorced, Widowed
- **Position** - Positions the person holds
- **Team** - Teams the person is part of
- **Organization Admin** — Whether the person is an organization admin

### Operators by Field

Different fields support different operators:

#### Age
- **Equals** - Age equals a specific number
- **Greater than** - Age is greater than a number
- **Less than** - Age is less than a number
- **Between** - Age is between two numbers (min and max)

#### Gender / Marital Status
- **Equals** - Matches exactly
- **Not equals** - Does not match

#### Position
- **Has** - Person has this position
- **Does not have** - Person does not have this position

#### Team
- **Is in** - Person is in this team
- **Is not in** - Person is not in this team

#### Org Admin
- **Is** - Person is an organization admin (true/false)

### Rule Examples

**Example 1: All Youth (Ages 13-18)**
- Field: Age
- Operator: Between
- Value: 13-18

**Example 2: All Worship Team Members**
- Field: Position
- Operator: Has
- Value: Worship Leader (or any worship position)

**Example 3: All Married Women**
- Rule 1:
  - Field: Gender
  - Operator: Equals
  - Value: Female
- Rule 2:
  - Field: Marital Status
  - Operator: Equals
  - Value: Married

**Example 4: All Non-Admin Team Leaders**
- Rule 1:
  - Field: Org Admin
  - Operator: Is
  - Value: No
- Rule 2:
  - Field: Position
  - Operator: Has
  - Value: Team Leader

### Adding Multiple Rules

To add multiple rules:

1. Create your first rule
2. Click **"Add Rule"**
3. Configure the new rule
4. Repeat as needed

<Info>
**Remember**: All rules use AND logic. A person must match ALL rules to be included. If you need OR logic (match any rule), create separate groups.
</Info>

### Editing Automatic Groups

To modify an automatic group:

1. Open the group
2. Click **"Edit"**
3. Add, modify, or remove rules
4. Click **Update Group** (web) or **Save** (mobile)

The group membership will automatically update based on the new rules.

## Viewing Groups

To view all groups:

1. Go to **People** → **Groups**
2. You'll see a list of all groups with:
   - Group name
   - Group type (Manual or Automatic)
   - Number of members

### Viewing Group Details

Click on any group to see:

- **Description** (if set)
- **Group Type** (Manual or Automatic)
- **Rules** (for automatic groups)
- **Teams** (for manual groups)
- **Members** - Complete list of all members

<Info>
**Note**: For automatic groups, the member list is calculated when you view the group. It reflects the current state of your organization based on the rules.
</Info>


## Best Practices

### When to Use Manual Groups

- **Fixed Committees** - Groups that don't change often
- **Special Teams** - Ad-hoc groups for specific projects
- **Small Groups** - Groups with just a few members
- **Custom Selections** - Groups that don't fit automatic rule patterns

### When to Use Automatic Groups

- **Age-Based Groups** - Youth, Seniors, etc.
- **Role-Based Groups** - All Worship Leaders, All Teachers, etc.
- **Team-Based Groups** - All members of specific teams
- **Dynamic Groups** - Groups that change as people join/leave teams or change roles

### Rule Design Tips

1. **Start Simple** - Begin with one rule, then add more if needed
2. **Test Rules** - After creating an automatic group, check the member list to ensure it matches your expectations
3. **Use Clear Names** - Name groups descriptively so others understand their purpose
4. **Document Rules** - Use the description field to explain complex rule logic
5. **Review Regularly** - Periodically review automatic groups to ensure rules are still accurate

## Editing Groups

**Web:** Click the **Edit** icon on the group card. Modify name, description, type, members, or rules → **Update Group**.

**Mobile:** Open the group → tap the **Edit** icon in the detail modal header → **Save**.

<Warning>
**Note**: Switching a group from automatic to manual (or vice versa) will clear the previous configuration. Make sure to note your rules or members before switching.
</Warning>

## Deleting Groups

**Web:** Click the **Delete** icon on the group card → confirm.

**Mobile:** Open the group → tap the **Delete** icon → confirm.

<Warning>
**Warning**: Deleting a group cannot be undone. All group data, including rules and member lists, will be permanently removed.
</Warning>

## Push Notifications

Send push notifications to group members:

**Web:**
1. Go to **People → Groups**
2. Click the **Push Notifications** tab (not inside an individual group)
3. Select one or more groups, enter your message, and send

**Mobile:**
1. Go to **More → Groups** (Owner/Admin)
2. Tap the **notification FAB** on the groups list
3. Select one or more groups, enter title and message, and send

Members receive the notification on their mobile devices if they have the ChurchBuddy app installed and notifications enabled.

## Permissions

To create, edit, or delete groups, you must be:

- **Owner** of the organization, OR
- **Organization Admin**

There is no separate Group Admin role in the current UI. Regular members and Team Admins do not have a Groups sub-tab.

## Troubleshooting

**Automatic group has no members:**
- Check that your rules are correct
- Verify that people in your organization actually match the rule criteria
- Remember that all rules use AND logic - a person must match ALL rules

**Manual group missing team members:**
- Team membership in groups is based on current team membership
- If the member list looks stale, refresh the page — new team members are included automatically

**Can't see Groups option:**
- You need **Owner** or **Organization Admin** permissions
- Access via **People → Groups** (web) or **More → Groups** (mobile)

**Group membership not updating:**
- For automatic groups, membership is calculated when you view the group
- Try refreshing or viewing the group again
- For manual groups, you must manually add/remove members
